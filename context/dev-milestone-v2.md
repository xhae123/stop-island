# 멈춰, 섬! — 웹 개발 마일스톤

> 작성: 김우진 (소프트웨어) · 2026-07-08
> 팝업 운영: 9/30 ~ 10/6 (7일간)

이 문서는 웹 개발 일정과 디자인팀 산출물 마감을 정리한 것입니다. 의견 있으면 편하게 말씀해주세요.

**큰 그림:**
- ~8월 중순: 기능 개발 완료
- 8월 하순: 디자인 시안 적용
- 9월 전반: QA / 팀 테스트
- 9/15~: 코드 프리즈, 팝업 준비에 집중

---

## 만들 화면

| # | 화면 | 내용 |
|---|---|---|
| 1 | 메인 | 오늘 방문 현황과 빈 자리 안내, 예약/방명록 진입 버튼 |
| 2 | 메뉴 선택 | 테이블 예약 또는 방명록 작성 선택, 참여 상점 목록 안내 |
| 3 | 영수증 인증 | 영수증 사진 촬영/업로드, 또는 이용한 상점 직접 선택 |
| 4 | 테이블 예약 | 6개 좌석(A1~B3) 중 빈 자리 선택 후 예약 확정 |
| 5 | 방명록 | 익명 후기 작성, 맛집/별점 태그, 스크롤하며 이전 후기 보기 |

이 5개 외에 추가로 필요한 화면 (일정에 반영됨):
- 로딩 중, 에러 발생 시 안내 화면
- 영수증 인증 성공/실패 화면
- 예약 완료 확인 화면

**참고 — 주요 설계 방향:**
- 영수증 인증은 사진 업로드 후 운영진이 확인하는 방식. 이용자가 상점을 직접 선택하는 방법도 함께 제공.
- 좌석 현황은 30초마다 자동 새로고침. 좌석이 6개뿐이라 충분.
- 관리자 기능은 좌석 열기/닫기, 방명록 삭제 정도로 최소화.

## 타임라인

### Phase 1 — 프로젝트 세팅 (7/8 ~ 7/13, 1주)

- 기술 스택 결정 (프론트엔드 프레임워크, 서버, 데이터베이스)
- 데이터베이스 설계 — 좌석, 예약, 상점, 영수증 인증, 방명록
- 도메인 확보, 서버 세팅, HTTPS 설정
- 배포 파이프라인 구성 (코드 올리면 자동으로 반영되게)
- 와이어프레임 기반 전체 화면 목록 + 상태 정리

> **디자인팀에게:** 개발 쪽은 뼈대를 세우는 중입니다. 시안 작업 진행 중이라면 아래 "디자인 체크포인트"를 참고해주세요.

---

### Phase 2 — 핵심 화면 개발 (7/14 ~ 8/3, 3주)

와이어프레임 기반으로 기능부터 만들고, 디자인 시안이 오면 입히는 방식.

**7/14 ~ 7/20 · 메인 + 메뉴 선택 + 영수증 인증**
- 메인 화면 Hero 영역, 프로젝트 소개 섹션
- 빈 자리 현황 카드, 오늘 방문자 수 표시
- "테이블 예약" / "방명록 작성" 선택 카드
- 참여 상점 목록 badge 표시
- 영수증 사진 촬영/업로드 UI, 카메라 연동
- 상점 직접 선택 (사진 없이도 인증 가능한 폴백)
- 인증 요청 서버 처리 — 이미지 저장, 인증 상태 관리
- 인증 성공/실패 결과 화면

**7/21 ~ 7/27 · 테이블 예약 + 방명록**
- 좌석 그리드 UI — A1~B3 6개 좌석, 상태 표시(빈 자리/사용 중/내가 선택)
- 좌석 선택 → 예약 확정 로직
- 30초 자동 새로고침 (다른 사람이 예약하면 반영)
- 예약 완료 확인 화면 — 좌석 번호, 이용 안내
- 방명록 작성 폼 — 내용 입력, 맛집 태그, 별점
- 방명록 목록 — 이전 후기 열람, 무한 스크롤
- 익명 처리, 작성 시간 표시("익명 방문자 · 10분 전")

**7/28 ~ 8/3 · 보조 화면 + 관리자 기능**
- 로딩 화면, 에러 화면 (네트워크 오류, 서버 오류 등)
- 인증 하루 1회 제한 처리
- 관리자 — 좌석 열기/닫기, 예약 현황 확인
- 관리자 — 방명록 삭제, 참여 상점 추가/수정
- 관리자 — 들어온 영수증 인증 요청 목록 확인/승인

> **디자인팀에게:** 이 기간에 시안이 도착하면 바로 반영합니다. 아직 없는 화면은 와이어프레임으로 먼저 만들어두니 걱정 않으셔도 됩니다.

---

### Phase 3 — 통합 + 안정화 (8/4 ~ 8/17, 2주)

**8/4 ~ 8/10 · 전체 연결 + 모바일 최적화**
- 전체 플로우 연결 — QR 스캔 → 메인 → 인증 → 예약 → 완료
- 전체 플로우 연결 — QR 스캔 → 메인 → 방명록 → 게시 완료
- QR 코드 생성, 테이블별 고유 QR 연동
- 모바일 최적화 — 다양한 화면 크기, iOS/Android 브라우저 호환
- 예외 상황 처리 — 중복 예약 방지, 예약 만료, 이미 인증한 사용자 재접근

**8/11 ~ 8/17 · 실제 데이터 + 버퍼**
- 실제 상점 데이터 입력 (막걸리계보, 조준영 목공방 등)
- 실제 좌석 배치도 반영
- 1차 버그 수정
- 밀린 작업 소화 버퍼

> **디자인팀에게:** 이 시점에서 기능이 거의 돌아갑니다. 링크 공유할 테니 직접 써보고 어색한 부분 알려주세요. **8/17까지 최종 시안 부탁드립니다.**

---

### Phase 4 — 디자인 적용 (8/18 ~ 8/31, 2주)

**8/18 ~ 8/24 · 시안 적용**
- 디자인 시스템 코드화 — 컬러(Island Yellow 등), 폰트, 간격 규칙
- 메인, 메뉴 선택, 영수증 인증 화면에 시안 적용
- 테이블 예약, 방명록 화면에 시안 적용
- 보조 화면(로딩/에러/완료)에 시안 적용
- 아이콘, 이미지 에셋 교체

**8/25 ~ 8/31 · 마감 조율**
- 관리자 화면 정리
- 디자인 적용 후 모바일 재점검, 미세 조정
- 디자인팀 피드백 반영
- 전체 최종 확인

> **디자인팀에게:** 이 기간이 디자인 적용 마지막 기회입니다. **8/17까지 최종 시안을 전달해주시면** 2주에 걸쳐 적용합니다. 이후 수정은 9월 QA에서 소소한 것만 가능합니다.

---

### Phase 5 — QA / 팀 테스트 (9/1 ~ 9/14, 2주)

**9/1 ~ 9/7 · 팀 테스트 + 피드백**
- 팀원 전원 테스트 — 영수증 찍고 → 인증하고 → 자리 잡고 → 후기 남기기
- 오탈자, 어색한 문구, 사소한 디자인 수정
- 현장 네트워크 테스트 — 실제 장소에서 QR 스캔, 모바일 데이터 환경 확인

**9/8 ~ 9/14 · 최종 점검**
- 야외 환경 대응 — 햇빛 아래 가독성, 저사양 기기 성능
- 운영진 관리 화면 교육
- 운영 매뉴얼 작성 (비개발자가 관리할 수 있도록)
- 비상 대응 플랜 (서버 다운, 인증 오류 시 수동 대응법)
- 최종 버그 수정

---

### Phase 6 — 코드 프리즈 + 팝업 준비 (9/15 ~ 9/29, 2주)

긴급 버그 외 개발 중단. 현장 설치, 콘텐츠 준비, 운영 리허설 등에 집중.

---

### 9/30 ~ 10/6 · 팝업 운영

현장 대응 및 모니터링.

## 디자인 체크포인트

| 시점 | 필요한 것 | 비고 |
|---|---|---|
| ~7월 말 | 화면별 디자인 방향 1차 공유 | 완성도 낮아도 OK. 없으면 와이어프레임으로 먼저 개발 |
| **~8/17** | **전체 화면 최종 시안 확정** | **디자인 마감. 8/18부터 코드에 적용 시작** |
