## 남은 태스크

### 1. 프론트엔드 프로젝트 초기화
- **뭘 하는지**: `frontend/`에 Svelte + Vite + Tailwind + svelte-spa-router 세팅. 빈 화면 5개 라우트.
- **절차**:
  - `npm create vite@latest frontend -- --template svelte`
  - Tailwind, svelte-spa-router 설치
  - 5개 라우트 스켈레톤 (`/`, `/menu`, `/verify`, `/reserve`, `/guestbook`)
- **완료 기준**: `npm run dev`로 5개 라우트 이동 가능

### 2. 백엔드 프로젝트 초기화
- **뭘 하는지**: `backend/`에 FastAPI + SQLite 세팅. DB 테이블 생성 + 시드 데이터.
- **절차**:
  - FastAPI, SQLAlchemy, uvicorn 설치
  - `context/db-schema.md` 기준 모델 정의 (SQLite 호환)
  - health check 엔드포인트 (`GET /api/health`)
  - API 스켈레톤 (경로만, 구현은 다음 주)
  - CORS 설정 (로컬 + Vercel origin)
- **완료 기준**: `uvicorn` 실행 → `/api/health` 응답, `/docs`에서 전체 엔드포인트 목록 확인

### 3. Docker Compose + OCI 배포
- **뭘 하는지**: 백엔드를 컨테이너화해서 OCI 기존 인스턴스에 올리기.
- **절차**:
  - `backend/Dockerfile` + `docker-compose.yml` 작성
  - SQLite 파일 + uploads 디렉토리 볼륨 마운트
  - OCI 인스턴스에 배포, nginx 프록시 설정 추가
- **완료 기준**: OCI에서 `curl localhost:8000/api/health` 응답

### 4. 도메인 + HTTPS
- **뭘 하는지**: 백엔드 API용 도메인 확보, nginx에 연결, Let's Encrypt HTTPS.
- **Tom 결정 필요**: 기존 보유 도메인 있으면 서브도메인 사용. 없으면 무료 서브도메인이나 저가 도메인.
- **절차**:
  - DNS 설정 (서브도메인 → OCI IP)
  - nginx에 server 블록 추가
  - `certbot --nginx -d api.<domain>`
- **완료 기준**: 외부에서 `https://api.<domain>/api/health` HTTPS 응답

### 5. Vercel 배포
- **뭘 하는지**: `frontend/`를 Vercel에 연결.
- **절차**:
  - Vercel에 레포 연결, Root Directory = `frontend/`
  - SPA rewrite 설정 (`vercel.json`)
  - `VITE_API_BASE_URL` 환경변수에 백엔드 HTTPS URL
- **완료 기준**: `*.vercel.app`에서 빈 화면 뜨고, 새로고침해도 라우트 유지

### 6. End-to-end 연동 확인
- **뭘 하는지**: Vercel 프론트 → OCI 백엔드 HTTPS 통신 확인.
- **절차**:
  - 메인 화면에서 `/api/health` fetch → 결과 표시
  - CORS 에러 없는지 확인
- **완료 기준**: 브라우저에서 Vercel URL 접속 → API 응답 표시, 콘솔 에러 없음
