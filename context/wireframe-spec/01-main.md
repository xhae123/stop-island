# 01 메인

> Figma node: 311:110 · 와이어프레임 이미지: `../figma/wireframes/web-wireframe-screen-1043-22__311-110.png`

## 화면 목적
"멈춰, 섬!" 모바일 웹 앱의 첫 진입 화면. 행궁동 공방거리 팝업 공간의 컨셉을 소개하고, 실시간 현황(빈 자리/방문자 수)을 보여준 뒤 "테이블 예약" 또는 "방명록 보기"로 사용자를 분기시킨다.

## 진입 경로
- QR코드 스캔(영수증 인증제 QR, 현장 배너 QR 등)으로 웹앱 최초 접속 시 랜딩 화면
- 앱 루트 경로(`/`)에 해당

## 레이아웃 구조
위에서 아래로 4개 영역:

1. **헤더 (Header/Nav)** — 다크 배경(#1a1700) 고정 상단바. 좌측 서비스명, 우측 메뉴 아이콘.
2. **히어로 영역 (Hero)** — 화면 폭 전체를 채우는 옐로우(#FFEE00) 배경 블록. 상단 라벨 + 타이틀(2줄) + 서브카피로 구성된 소개 카피 영역.
3. **현황 카드 영역 (Status)** — "지금 현황" 섹션 타이틀 아래 2개의 통계 카드(빈 자리 / 오늘 방문)가 가로로 나란히 배치.
4. **액션 영역 (CTA)** — 세로로 쌓인 버튼 2개: Primary(테이블 예약하기), Secondary(방명록 보기).

## 컴포넌트 상세

### 1. Header
- **타입**: 고정 상단 네비게이션 바(sticky top)
- **배경색**: `#1a1700`
- **좌측**: 서비스 로고/타이틀 텍스트 "로컬코 × 행궁동" (흰색 텍스트, semi-bold)
- **우측**: 원형 배경의 햄버거 메뉴 아이콘 버튼 (≡). 탭 시 사이드 메뉴/네비게이션 드로어 오픈 (전체 5개 화면으로 이동 가능한 메뉴로 추정 — 01 메인/02 메뉴 선택/03 영수증 인증/04 테이블 예약/05 방명록)
- **상태바 영역**: 모바일 상태바(9:41, 네트워크/배터리 아이콘) — 실제 구현 시 시스템 UI이므로 개발 대상 아님, 와이어프레임 표기용

### 2. Hero 영역
- **타입**: full-width 섹션, 배경 fill
- **배경색**: `#FFEE00` (메인 컬러)
- **구성 (위→아래)**:
  - 라벨 텍스트: "공방거리 팝업 휴식처" (작은 폰트, 서브 라벨)
  - 헤드라인 텍스트: "행궁동 사람들의 공유 다이닝 룸" (2줄, 굵은 대형 폰트, 화면 핵심 카피)
  - 서브카피 텍스트: "동네 상점 영수증 하나로 테이블을 예약하세요" (2줄, 중간 굵기)
- **인터랙션**: 없음 (정적 소개 텍스트 블록)

### 3. 현황 카드 영역 (Status Cards)
- **섹션 타이틀**: "지금 현황" (좌측 정렬, 작은 라벨)
- **카드 1 — 빈 자리**
  - 라벨: "빈 자리"
  - 값: "4석" (대형 숫자 + 단위)
  - 배경: 연한 그레이 카드, 라운드 코너
- **카드 2 — 오늘 방문**
  - 라벨: "오늘 방문"
  - 값: "38명" (대형 숫자 + 단위)
  - 배경: 연한 그레이 카드, 라운드 코너
- **레이아웃**: 2개 카드 가로 50:50 분할, 카드 사이 간격(gutter) 존재
- **데이터 소스**: 실시간 API polling (30초 간격) — 아래 "데이터" 섹션 참고
- **상태**: 로딩 중에는 skeleton 또는 "—" placeholder 표시, API 실패 시 마지막 캐시값 유지 + 에러 아이콘(선택)

> 참고: 이미지 상 "지금 현황" 라벨 옆에 "visualize: Lokalco wireframe briefing"이라는 텍스트 말풍선이 겹쳐 있으나, 이는 Figma 코멘트/주석 오버레이로 판단됨 (실제 UI 요소 아님). 개발 대상에서 제외.

### 4. CTA 버튼 영역
- **버튼 1 — 테이블 예약하기 (Primary)**
  - 텍스트: "테이블 예약하기"
  - 스타일: `primary` 버튼 (와이어프레임 상 흰 배경 + 검은 테두리/텍스트, full-width)
  - 액션: 탭 시 `/reserve` 라우트로 이동 (04 테이블 예약 화면, 혹은 02 메뉴 선택 → 03 영수증 인증 플로우로 진입할 가능성 있음 — 라우팅 확정은 Tom 확인 필요)
- **버튼 2 — 방명록 보기 (Secondary)**
  - 텍스트: "방명록 보기" (아이콘 포함 — 책/문서 아이콘)
  - 스타일: outline 버튼, full-width
  - 액션: 탭 시 `/guestbook` 라우트로 이동 (05 방명록 화면)

## 데이터

- `GET /api/status`
  - 응답: `{ available_seats: number, today_visitors: number }`
  - **폴링**: 30초 간격 재요청 (실시간 현황 갱신 목적)
  - 화면 진입 시 최초 1회 즉시 fetch 후 polling 시작
  - 화면 이탈(unmount) 시 polling 인터벌 정리 필요

## 상태

- **로딩**: 최초 진입 시 현황 카드 값 로딩 중 — skeleton UI 또는 dash("-") 표시
- **정상**: `available_seats`, `today_visitors` 값 정상 표시
- **에러**: API 실패 시 마지막 성공 값 유지, 필요 시 작은 에러 인디케이터 노출 (화면 전체를 블로킹하는 에러 상태는 없음 — 히어로/CTA는 항상 노출)
- **빈 자리 0석**: "빈 자리 0석" 등 값 자체는 정상 표시하되, 예약 버튼 탭 시 04 화면에서 마감 안내 처리 (이 화면 자체에서 버튼을 비활성화할지는 Tom 확인 필요)

## 이동

- 헤더 우측 메뉴 아이콘 탭 → 전체 화면 네비게이션 메뉴 오픈 (01~05 화면 목록)
- "테이블 예약하기" 버튼 탭 → `/reserve` (예약 플로우 시작, 02 메뉴 선택 또는 04 테이블 예약으로 연결 — 플로우 순서 Tom 확인 필요)
- "방명록 보기" 버튼 탭 → `/guestbook` (05 방명록 화면)

## 개발 노트

- **메인 컬러**: `#FFEE00` — Hero 영역 배경 fill에 사용
- **헤더/네비 배경색**: `#1a1700` — 다크 배경, 상단 고정 헤더에 적용
- **Hero 영역**: full-width, `#FFEE00` fill. 반응형 시에도 좌우 여백 없이 화면 폭 전체 채움
- **현황 카드**: 실시간 API 연동 + 30초 polling 필수 구현 사항 (정적 텍스트 아님)
- **CTA 버튼**: `primary` 스타일 클래스/컴포넌트 사용 (디자인 시스템 상 정의된 primary 버튼 재사용)
- **라우팅**: `/reserve`, `/guestbook` 2개 경로가 이 화면에서 직접 연결됨 (개발 스펙 패널에 명시)
- **API**: `GET /api/status` → `available_seats`, `today_visitors` 반환. 다른 필드 없음 (스펙 패널 원문 기준)
- 상태바(9:41 등)는 iOS 목업 표기이며 실제 웹앱 구현 대상 아님
- 색상/타이포 등 세부 디자인 토큰은 디자인 팀(김다인, 김민진, 황설휘) 확정 필요 — 이 문서는 구조/인터랙션/데이터 스펙에 한정
