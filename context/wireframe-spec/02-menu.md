# 02 메뉴 선택

> Figma node: 311:109 · 와이어프레임 이미지: `../figma/wireframes/web-wireframe-screen-1043-28__311-109.png`

## 화면 목적

메인 화면(01) 진입 직후, 사용자가 "무엇을 하러 왔는지"를 먼저 선택하게 하는 분기 화면. 두 가지 목적 중 하나를 선택한다.

1. **테이블 예약** — 영수증 인증 후 좌석(테이블)을 예약하는 흐름으로 진입
2. **방명록 작성** — 후기·맛집 정보를 공유하는 흐름으로 진입

이 화면 자체는 데이터를 생성하지 않고, 이후 흐름(03/04/05)으로 라우팅하는 역할만 한다.

## 진입 경로

- **01 메인** 화면에서 진입. 메인 화면의 CTA(예: "시작하기" 버튼 또는 QR 스캔 후 랜딩)를 통해 이 화면으로 이동한다고 추정. 정확한 트리거는 01 메인 화면 스펙에서 재확인 필요.
- 뒤로가기(←)로 01 메인으로 복귀 가능.

## 레이아웃 구조

모바일 뷰포트(375~390px 기준 폭) 세로 스크롤 단일 컬럼 레이아웃. 상단부터 순서대로:

```
[상태바 9:41]
[헤더: 뒤로가기 버튼(←) + 타이틀 "무엇을 하시겠어요?"]  ← 배경색 Island Yellow(#FFEE00)
[본문 영역: 배경 흰색]
  [섹션 라벨: "이용 목적을 선택하세요"]
  [메뉴 카드 1: 테이블 예약]  ← 선택된(강조) 상태로 렌더링
  [메뉴 카드 2: 방명록 작성]  ← 비선택 상태
  [참여 상점 안내 박스]
    [섹션 라벨: "참여 상점"]
    [상점 배지 리스트 (가로 스크롤 또는 wrap)]
```

- 헤더는 Island Yellow 배경 위에 검정 텍스트, 뒤로가기 아이콘은 좌측 정렬, 타이틀은 중앙 정렬.
- 본문은 흰 배경, 카드 2개가 세로로 쌓인 리스트 형태(수직 스택, gap 존재).
- 참여 상점 박스는 카드 리스트 하단에 별도 회색조 배경 섹션으로 분리.

## 컴포넌트 상세

### 1. Header
- 배경: `#FFEE00` (Island Yellow)
- 뒤로가기 버튼: 좌측 아이콘(←), 탭 시 이전 화면(01 메인)으로 이동
- 타이틀: "무엇을 하시겠어요?" — 중앙 정렬, bold, 검정 텍스트

### 2. 섹션 라벨
- 텍스트: "이용 목적을 선택하세요"
- 회색조 텍스트(보조 텍스트 스타일), 카드 리스트 상단에 위치

### 3. 메뉴 카드 (MenuOptionCard) — 공통 구조
각 카드는 다음 요소로 구성된 selectable list item:
- 좌측: 아이콘 원형 배지 (icon badge, 원형 배경)
- 중앙: 타이틀(bold) + 서브텍스트(설명, 회색조 보조 텍스트)
- 우측: chevron(`>`) 아이콘 — 다음 화면으로 이동 가능함을 나타내는 어포던스

#### 카드 1 — 테이블 예약 (선택 상태로 표시됨)
- 아이콘: 테이블/좌석 아이콘, 아이콘 배지 배경 `#FFEE00`(Island Yellow)
- 타이틀: "테이블 예약"
- 서브텍스트: "영수증 인증 후 좌석 선택"
- 카드 전체 배경: 연한 노란빛 틴트(옅은 크림/옅은 옐로우, Island Yellow의 저채도 tint)
- **선택 상태 스타일 (개발 스펙 패널 명시)**: `border: 2px solid #FFEE00`
- 탭 시 이동: **03 영수증 인증** 화면으로 push

#### 카드 2 — 방명록 작성 (비선택 상태)
- 아이콘: 펜/서명 아이콘, 아이콘 배지 배경은 중립 회색(비강조)
- 타이틀: "방명록 작성"
- 서브텍스트: "후기 · 맛집 공유"
- 카드 전체 배경: 흰색(비강조)
- **비선택 상태 스타일 (개발 스펙 패널 명시)**: `border: 1px solid var(--border)`
- 탭 시 이동: **05 방명록** 화면으로 push
  - (참고: 플로우차트 페이지 순서상 04=테이블 예약, 05=방명록. "영수증 인증(03)"은 테이블 예약 흐름 전용 선행 단계이며, 방명록 작성은 영수증 인증을 거치지 않고 05로 바로 진입하는 것으로 보임. 01 메인/03/04/05 스펙과 교차 확인 필요)

> 두 카드는 라디오 그룹처럼 상호배타적 선택 상태를 가지는 것으로 보이나(하나만 강조 표시), 실제로는 탭 즉시 다음 화면으로 push되는 내비게이션 아이템일 가능성이 높다. 즉 "선택 상태"는 hover/active/default 강조가 아니라 **탭 이벤트 발생 시 표시되는 순간적 강조 스타일**로 구현한다 (아래 상태 섹션 참조).

### 4. 참여 상점 안내 박스 (ParticipatingShopsPanel)
- 배경: 옅은 회색 (카드 리스트와 구분되는 섹션)
- 섹션 라벨: "참여 상점"
- 상점 배지(badge) 리스트: 가로 나열
  - 배지 예시: "카페 A", "서점 B", "음식점 C" — Island Yellow 배경의 pill/chip 형태
  - 마지막에 "+12개" 텍스트로 나머지 상점 개수 표시 (배지 아닌 plain text, 회색)
- **개발 스펙 명시**: 데이터 소스는 `GET /api/shops` — "API list, badge 렌더". 참여 상점 목록을 API로 불러와 배지 형태로 렌더링.
- 이 박스 자체는 탭 가능한 인터랙션이 없는 것으로 보임 (정적 정보 표시). 단, "+12개" 텍스트나 배지 탭 시 상점 상세/목록 화면으로 이동하는 기능은 와이어프레임에 명시되어 있지 않음 — 필요 시 Tom 확인 후 확장.

### 5. 하단 페이지 인디케이터
- 화면 최하단에 점(dot) 5개 — 전체 플로우(01~05) 중 현재 위치(2번째)를 표시하는 프로그레스 인디케이터로 추정. 실제 화면 컴포넌트가 아니라 Figma 프레젠테이션용 네비게이션 표시일 가능성 높음 — **구현 대상 아님** (개발 노트 참고).

## 데이터

### API: 참여 상점 목록
```
GET /api/shops
```
- **응답**: 참여 상점 목록 배열 반환 ("참여 상점 목록 반환")
- **예상 응답 스키마** (명시 안 됨, 추정치 — Tom 확인 필요):
  ```json
  {
    "shops": [
      { "id": "string", "name": "카페 A" },
      { "id": "string", "name": "서점 B" },
      { "id": "string", "name": "음식점 C" }
    ],
    "totalCount": 15
  }
  ```
- 화면에는 상위 N개(와이어프레임 예시 기준 3개)만 배지로 렌더링하고, 나머지는 `totalCount - N`을 "+{n}개"로 표시.
- 이 화면에서 별도로 사용자 입력을 서버로 전송하는 액션은 없음 (읽기 전용 화면).

## 상태

- **선택 상태 (selected)**: `border: 2px solid #FFEE00`. 카드 탭 시 또는 활성 카드에 적용.
- **비선택 상태 (unselected)**: `border: 1px solid var(--border)` — CSS 커스텀 프로퍼티 `--border`를 디자인 토큰으로 사용 (프로젝트 전역 테마 변수, 값은 디자인 시스템에서 정의된 중립 회색 예상).
- 두 카드 중 하나가 항상 selected 스타일로 보이는 와이어프레임 표현은, 실제 구현에서는 다음 중 하나로 해석 가능 (구현 전 Tom 확인 필요):
  1. 단순히 "선택 가능함/주목도가 높은 카드"를 보여주기 위한 정적 목업 표현이며, 실제로는 두 카드 모두 default(비선택) 상태로 시작하고 탭 시에만 selected 스타일이 잠깐 적용된 후 즉시 push
  2. 실제로 라디오 선택 상태를 유지하다가 별도의 "다음" 버튼으로 진행 (단, 이 와이어프레임에는 "다음" 버튼이 없으므로 가능성 낮음)
  - **기본 구현 가정**: 각 카드는 즉시 이동형 리스트 아이템(list navigation item)이며, `selected` 클래스는 탭 시 press/active 피드백 용도로만 사용한다.
- 로딩 상태: 참여 상점 API 응답 대기 중 배지 영역에 스켈레톤 또는 로딩 placeholder 필요 (와이어프레임에 명시 안 됨, 구현 시 추가 권장).
- 에러 상태: API 실패 시 참여 상점 섹션을 숨기거나 "참여 상점 정보를 불러오지 못했습니다" 등 폴백 텍스트 표시 (와이어프레임에 명시 안 됨, 구현 시 추가 권장).

## 이동

라우팅 규칙 (개발 스펙 패널 명시): **"선택 → 다음 화면 push"**

| 트리거 | 대상 화면 |
|---|---|
| 헤더 뒤로가기(←) 탭 | 01 메인으로 back navigation |
| "테이블 예약" 카드 탭 | 03 영수증 인증 화면으로 push |
| "방명록 작성" 카드 탭 | 05 방명록 화면으로 push |

- 이동 방식은 SPA 라우터 기준 `push` (뒤로가기 시 이 메뉴 화면으로 복귀 가능해야 함). 브라우저/앱 히스토리 스택에 현재 화면을 유지한 채 다음 화면을 쌓는다.

## 개발 노트

- 색상 토큰: Island Yellow `#FFEE00`, 카드 선택 테두리 2px, 비선택 테두리 1px `var(--border)`. 프로젝트 전역 디자인 시스템(CLAUDE.md 컬러: Island Yellow / Pause White / Flow Black)과 일치하게 CSS 변수로 관리할 것.
- 아이콘: 테이블 예약 아이콘(좌석/테이블 모티브), 방명록 아이콘(펜/서명 모티브) — 별도 아이콘 에셋 필요, 디자인팀(김다인/김민진/황설휘) 확인 후 최종 SVG 반영.
- 하단 점(dot) 인디케이터 5개는 Figma 프레젠테이션 캔버스 상의 화면 목록 네비게이션으로 판단되며, 실제 앱 화면에는 포함하지 않는다. 만약 실제 온보딩성 진행률 표시가 의도된 것이라면 Tom 확인 필요.
- 상단 탭바("01 메인 / 02 메뉴 선택 / 03 영수증 인증 / 04 테이블 예약 / 05 방명록")는 Figma 편집용 네비게이션이며 실제 서비스 화면 구성 요소 아님 — 구현 대상 아님.
- 참여 상점 배지 개수/순서 로직(어떤 3개를 우선 노출할지 — 최신순/랜덤/제휴 우선순위 등)은 명시되어 있지 않음. 백엔드 정렬 기준 Tom 확인 필요.
- 접근성: 카드 전체가 탭 영역(전체 row가 button/link 역할)이 되도록 구현. chevron 아이콘은 장식용(aria-hidden)으로 처리하고 카드 자체에 적절한 role/label 부여.
- 이 화면은 상태를 서버에 저장하지 않는 순수 클라이언트 라우팅 화면이므로, 전역 상태 관리(예: 선택한 목적)를 다음 화면에 전달할 필요가 있다면 라우터 파라미터 또는 로컬 상태로 전달 (예: `navigate('/receipt-auth', { purpose: 'table-reservation' })`). 목적값을 백엔드로 넘겨야 하는지는 03/04 화면 스펙에서 확인 필요.
