# 멈춰, 섬!

이 레포는 "멈춰, 섬!" 프로젝트의 **모노레포**다. 기획·디자인 맥락 + 웹 서비스 코드 + AI 에이전트 하네스를 한 곳에서 관리한다.

## 핵심 컨셉

**"멈춰, 섬!"** = "멈춰 서다" + "섬(island)"

도심 속 가장 작은 섬. 돗자리/조립식 목가구/파라솔을 유휴 공간에 배치해서 "지나가는 거리"를 "머무르는 거리"로.

**3대 솔루션:**
1. **오픈 스트릿 라운지** — 시그니처 돗자리, 캠핑 박스 테이블, 평상. 조준영 목공방 협업 조립식 모듈형 가구
2. **영수증 인증제** — F&B 영수증 = 공간 이용 티켓. QR→웹→인증→예약→방명록
3. **"길막"(길거리 막걸리)** — 막걸리계보 협업. 공방 제작 잔 대여→골목 막걸리 피크닉→반납

## 웹 서비스 — 주요 기능

QR 코드를 찍으면 열리는 모바일 웹. 앱 설치 없이 브라우저에서 동작.

| 화면 | 기능 | 비고 |
|---|---|---|
| 메인 | 빈 자리 현황, 방문자 수, CTA | `GET /api/status` |
| 메뉴 선택 | 예약/방명록 분기, 참여 상점 목록 | `GET /api/shops` |
| 영수증 인증 | 사진 업로드 + 상점 직접 선택 폴백 | `POST /api/verify` · 운영진 수동 확인 방식 |
| 테이블 예약 | A1~B3 좌석 그리드, 예약 확정 | `GET /api/seats` `POST /api/reserve` · 30초 폴링 |
| 방명록 | 익명 후기, 맛집 태그, 무한스크롤 | `POST /api/guestbook` `GET /api/guestbook` |
| 관리자 | 좌석 열기/닫기, 방명록 삭제, 인증 승인 | 운영진 전용 |

## 작업 방식 — GitHub 이슈 기반

> **모든 작업은 GitHub 이슈로 관리한다.** 이슈 없이 코드를 짜지 않는다.

### 흐름
1. 주차별 플랜(`context/plans/week-XX.md`)에서 태스크 목록 확인
2. 각 태스크는 GitHub 이슈로 존재 (`[P{phase}-{번호}]` 형식)
3. 이슈를 받으면 → 이슈 본문의 "할 일" 체크리스트 순서대로 수행
4. 완료되면 이슈 닫기

### 이슈 컨벤션
- 제목: `[P{phase}-{번호}] 태스크 제목` (예: `[P1-1] 프론트엔드 초기화`)
- 라벨: `phase-N`, `frontend`/`backend`/`infra`
- 본문: 목표, 할 일(체크리스트), 완료 기준, 의존성
- 레포: `xhae123/stop-island` (private)
- **gh 명령어 prefix**: 프로젝트 로컬 CLAUDE.md 참조 (`~/.claude/projects/` 하위)

### 세션 시작 시
1. `gh issue list --state open`으로 현재 열린 이슈 확인
2. 의존성이 풀린 가장 앞 번호 이슈부터 수행
3. 이슈 본문을 읽고 바로 실행

## AI 작업 지침

- **Figma가 단일 진실.** `context/figma/`는 스냅샷이라 썩을 수 있음 — 의심되면 API로 다시 읽어라
- **김우진 = 소프트웨어 1인 개발.** 웹 전체(프론트+백+인프라)를 담당
- **디자인 결정은 디자인 팀원(김다인, 김민진, 황설휘)의 영역.** 개발 쪽에서 art-direct 하지 않는다
- **마일스톤 기준으로 작업.** `context/dev-milestone-v2.md` 참고
- **화면 구현 시 wireframe-spec 참조.** `context/wireframe-spec/0X-xxx.md`가 상세 스펙
